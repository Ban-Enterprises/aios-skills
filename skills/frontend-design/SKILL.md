# Frontend Design

> Produkcyjny, dopracowany frontend — bez „generycznego AI”.

## Co robi

- Buduje realny, działający kod UI z wysoką dbałością o detal estetyczny i odważne decyzje projektowe.
- Korzysta z biblioteki **9 kurowanych przykładów** (od neo-brutalizmu, przez dark/hacker, po edytorial-luxury i wielostronicowy SaaS) — można je mieszać i adaptować.
- Metodyka: dobierz 2–3 najbliższe przykłady, przeczytaj ich pełne źródła, zinternalizuj system projektowy, dopiero potem buduj pod markę użytkownika.
- Celowo unika „AI slop” — generyczne layouty, przewidywalne gradienty, sztampowe komponenty.

## Kiedy się aktywuje

Gdy budujesz lub stylujesz cokolwiek webowego: strona, landing, dashboard, komponent
React/Vue/Svelte, HTML/CSS, mockup, prototyp.

## Jak działa (skrót)

```
1. Przejrzyj katalog przykładów → wybierz 2–3 najbliższe estetyce celu
2. Przeczytaj pełne źródła tych przykładów (mapa ≠ teren)
3. Buduj z głęboko przyswojonymi wzorcami, adaptując do marki i treści
```
